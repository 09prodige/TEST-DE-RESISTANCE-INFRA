# Project Management — Scrum/Kanban

## Board GitHub Projects
Colonnes : `Backlog → Sprint → In Progress → Review → Done`

## Sprints
- **Sprint 1** — Recon (subdomain, DNS, WHOIS, ports)
- **Sprint 2** — Web Analysis (headers, SSL, crawler, fingerprint)
- **Sprint 3** — Vulnerability Scanning (SQLi, XSS, CSRF, OWASP)
- **Sprint 4** — Reporting (HTML, JSON, PDF, CVSS scoring)

## Cérémonies
- Sprint Planning : sélection + estimation issues
- Daily : vérification progression agents
- Sprint Review : validation livrables
- Rétro : optimisation processus + tokens

## Définition of Done
- [ ] Critères d'acceptation validés
- [ ] Tests QA passés
- [ ] Audit sécurité validé (cybersecurity-engineer)
- [ ] PR mergée sur `main`
- [ ] Issue fermée sur board
